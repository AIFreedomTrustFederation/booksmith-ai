# Publishing Operations Playbook

This playbook describes how AI Freedom Trust Publishing can move a book from manuscript to global marketplace availability while keeping Booksmith AI local-first, inspectable, and author-controlled.

## Operating Model

AI Freedom Trust Publishing should operate as a professional independent publisher/imprint with a repeatable production pipeline.

The practical operating stack is:

1. Booksmith AI for manuscript, editorial, metadata, launch assets, and export preparation.
2. AI Freedom Trust Publishing as the publisher/imprint of record.
3. Publisher-owned ISBNs for professional metadata control.
4. Print-on-demand providers for paperback and hardcover fulfillment.
5. Ebook distribution for Kindle, EPUB, and wide digital retailers.
6. Direct sales through the Federation's own catalog when ready.

## Recommended Distribution Strategy

Use a hybrid distribution approach:

### Amazon KDP

Use for Kindle ebooks and Amazon-native print editions when direct Amazon control is useful.

Advantages:

- strong Amazon discoverability
- Kindle ecosystem access
- simple upload workflow
- fast file updates
- direct royalty reporting

### IngramSpark Or Equivalent Wholesale POD

Use for bookstore, library, and expanded wholesale reach.

Advantages:

- bookstore and library ordering channels
- global print-on-demand network
- professional wholesale metadata
- hardcover and paperback distribution options

### Draft2Digital Or Equivalent Ebook Aggregator

Optional for wide ebook distribution to retailers outside Amazon.

Advantages:

- one upload to multiple ebook retailers
- simplified metadata management
- useful for Kobo, Apple Books, Barnes & Noble ebook channels, and library-adjacent channels depending on distributor support

### Direct Sales

Use the Federation's own site for:

- signed copies
- PDF editions
- bundles
- workbooks
- courses
- member libraries
- special editions
- donations or pay-what-you-can releases when appropriate

Direct sales should not replace marketplace distribution at first. It should become the sovereign catalog and relationship layer.

## Book Production Pipeline

### 1. Intake

- title
- subtitle
- author name
- author legal/contact record
- manuscript source
- book category
- intended audience
- rights status
- AI-assistance disclosure preference
- cover/art status
- desired formats
- release timeline

### 2. Editorial Preparation

- structure review
- chapter list
- manuscript cleanup
- continuity pass
- style/voice pass
- author approval
- final manuscript lock

### 3. Metadata Preparation

- publisher name
- imprint name
- ISBN per format
- BISAC/category candidates
- keywords
- short description
- long description
- author bio
- series metadata
- age range, if applicable
- contributor roles
- price by format
- publication date
- territory rights

### 4. Interior Production

- Markdown export
- EPUB export
- LaTeX source export
- print-ready PDF
- trim size
- margins
- headers/footers
- fonts
- images
- tables/figures
- citations
- index/glossary if applicable

### 5. Cover Production

- ebook cover
- paperback wrap
- hardcover wrap or dust jacket when needed
- spine width based on final page count and paper type
- barcode area
- publisher logo placement
- imprint mark
- back cover copy

### 6. Proofing

- automated checks
- human editorial proof
- PDF proof
- EPUB proof
- print proof order
- correction log
- final approval

### 7. Distribution Upload

- upload files
- enter metadata
- set territories
- set pricing
- order proof
- approve proof
- verify live listings
- verify publisher/imprint metadata

### 8. Launch

- launch page
- email sequence
- social posts
- press/review packet
- author page
- catalog entry
- launch date checklist

### 9. Post-Launch Stewardship

- royalty tracking
- sales reporting
- file revision history
- edition tracking
- review monitoring
- errata log
- rights/licensing inquiries
- author payout records

## Required Records Per Title

Each published title should have a structured record containing:

- book ID
- title and subtitle
- author/contributor records
- publisher/imprint
- ISBNs by format
- copyright holder
- license statement
- source project hash or version
- export package version
- cover file version
- print PDF version
- EPUB version
- marketplace upload dates
- proof approval date
- publication date
- price history
- royalty terms
- rights notes
- AI provenance notes

## Quality Standard

Do not publish directly from raw AI output. Every released title should pass through human review, editorial approval, proofing, metadata review, and distribution verification.

Booksmith can accelerate the work, but AI Freedom Trust Publishing should be known for trustworthy, inspectable, human-approved books.
